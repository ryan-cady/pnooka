# PNOOKA — Claude Context

Minimal dot-matrix Pebble watchface inspired by Nooka watches. "The P is silent."

## What it does

- **4×3 dot grid** shows the hour (fills column-by-column, top-to-bottom)
- **60-segment bar** below the dots shows the minute (one segment = one minute)
- **AM/PM indicator** bottom-right, with a lit dot for the active period
- **Date** bottom-left (MM/DD/YY), toggleable
- All colors fully customizable via Clay settings page in the Pebble app

## Platforms

| Platform | Device | Display |
|---|---|---|
| Basalt | Pebble Time | 144×168, color |
| Emery | Pebble Time 2 / Time Steel | 200×228, color |
| Aplite | Pebble Classic / Pebble Steel | 144×168, B&W |
| Diorite | Pebble 2 | 144×168, B&W |

B&W platforms use black/white defaults (`PBL_BW` macro). The color picker on color platforms (Basalt/Emery) shows the full 64-color Pebble palette. On B&W platforms, `allowGray: true` in Clay config gives black/gray/white; without it only black/white are available.

## Project structure

```
src/c/main.c           — All watch-side C code (drawing, settings, tick handler)
src/pkjs/index.js      — PebbleKit JS entry point; Clay init + ready handler
src/pkjs/config.js     — Clay settings config (colors + date toggle)
package.json           — App manifest (UUID, messageKeys, platforms)
wscript                — Waf build script
p5 sketch/             — Original p5.js prototype used to design the layout
```

## Build & install

```bash
pebble build                          # outputs build/pnooka.pbw
pebble install --phone <phone-ip>     # phone IP from Pebble app → Settings → Developer Mode
```

## Message keys

Defined in `package.json` → auto-generated by SDK at build time:

| Key | Value | Type |
|---|---|---|
| BG_COLOR | 10000 | uint32 (RGB24) |
| FILLED_COLOR | 10001 | uint32 (RGB24) |
| EMPTY_COLOR | 10002 | uint32 (RGB24) |
| SHOW_DATE | 10003 | uint32 (0 or 1) |

Colors are sent as 24-bit RGB integers by Clay and converted to Pebble's 8-bit ARGB format via `rgb24_to_argb8()` in `main.c`. The C side persists all settings with `persist_write_int` so they survive watch restarts.

## Settings flow

1. User taps gear icon → `showConfiguration` → Clay opens settings page
2. User saves → `webviewclosed` → Clay calls `Pebble.sendAppMessage` → `inbox_received_handler` on watch
3. On every app start (`ready` event) → `index.js` re-sends stored Clay settings from `localStorage` to the watch

The `ready` re-send (step 3) is critical — without it, settings are lost on reconnect/restart because Clay doesn't do this automatically.

## Known constraints

- Inbox buffer is 128 bytes (`app_message_open(128, 0)`) — sufficient for 4 keys (~45 bytes total)
- Outbox is 0 bytes; the watch never sends messages, only receives
- The p5 sketch in `p5 sketch/` is a legacy design prototype, not part of the build
- `build/` and `node_modules/` are gitignored
